# Sidebar Project Items: Fix X Button Behavior

## Context / Problem Statement

When clicking the X button on an open workspace item in the sidebar, the project is removed from `openWorkspaces` but **remains visible** because it stays in the `recentProjects` list and gets re-rendered as a "recent project" item. There is no way to remove a project from the recent list.

The user wants:
1. X on **open workspaces** → truly close the workspace (remove from sidebar + call backend `closeWorkspace`), but keep in recent projects.
2. A separate X button on **recent projects** → remove from the recent projects list only.

## Key Findings (Prova Real)

- `removeFromSidebar` (line 456-463) only filters `openWorkspaces` + saves — never touches `recentProjects`.
- `closeOpenWorkspace` (line 192-198) already does `removeFromSidebar`'s job **plus** calls `closeWorkspace` in the backend — but it is **never called** by the X button.
- The X button in the template (line 452-462) calls `removeFromSidebar(proj)`.
- Recent projects render at line 468-483 with no X/delete button.
- Translations file likely has a key for `app.sidebar.closeWorkspace` already.

## Changes

### Change 1: Wire X button on open workspaces to `closeOpenWorkspace`
- **File:** `src/App.tsx`, lines 452-462
- **Mutation:** Change the X button's `onClick` handler from `removeFromSidebar(proj)` to `closeOpenWorkspace(proj)`
- **Why:** `closeOpenWorkspace` removes from `openWorkspaces` (so item disappears from sidebar) AND calls the backend `closeWorkspace` to properly close it. The item stays in `recentProjects` so it's still showable.
- **Constraints:** Safe to call multiple times; if already closed, `closeWorkspace` is a no-op on the backend.

### Change 2: Add X button to recent project items
- **File:** `src/App.tsx`, ~lines 468-483
- **Mutation:** Change the recent project `<button>` wrapper to a `<div>` (like open workspaces), and add a nested close button that removes the project from `recentProjects` + calls `saveRecent`.
- **Why:** Gives users a way to clean up the recent projects list, matching the requested two-button UX.
- **Details:**
  - Add a local function `removeRecent(proj)` that filters `recentProjects` and calls `saveRecent`.
  - The recent item becomes a `<div class="group flex ...">` containing the existing click-to-open button + the new X button.
  - X button calls `removeRecent(proj)` with `e.stopPropagation()`.

### Change 3: Clean up `removeFromSidebar`
- **File:** `src/App.tsx`, ~lines 456-463
- **Mutation:** Delete the `removeFromSidebar` function entirely once nobody calls it.
- **Why:** Dead code.
- **Constraints:** Ensure no other references exist (grep for the name).
