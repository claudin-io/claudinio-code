# Adicionar versão do app ao lado do logo Claudinio Code

## Contexto
O header tem uma pill arredondada no canto superior esquerdo com o ícone + "Claudinio **Code**". Queremos adicionar a versão atual do app (`v0.1.5`) logo após "Code", dentro da mesma pill, como um footnote discreto.

## Decisões de Design (confirmadas com o usuário)
- **Posição**: Dentro da pill existente, inline após "Code"
- **Formato**: `v0.1.5` (prefixo "v" minúsculo)
- **Separador**: `·` (middot / bullet) antes da versão
- **Cor da fonte**: `text-ink-faint`
- **Peso da fonte**: `font-extralight` (weight 200)
- **Tamanho**: ~9px (hairline footnote, ~70% do texto `text-[13px]` do logo)

## Solução

### Como expor a versão no frontend
Usar Vite `define` para injetar a versão do `package.json` como `import.meta.env.VITE_APP_VERSION` — zero dependências novas, zero comandos Tauri adicionais.

### Mudanças

1. **`vite.config.ts`** — Adicionar `define: { 'import.meta.env.VITE_APP_VERSION': JSON.stringify("...") }` lendo do `package.json`.

2. **`src/App.tsx`** (linha ~582) — Após o `<span class="text-accent">Code</span>`, adicionar:
   ```tsx
   <span class="ml-1 text-[9px] font-extralight text-ink-faint">· v{APP_VERSION}</span>
   ```

Nada mais muda — a pill existente já tem padding (`px-3 py-1`) que acomoda o extra sem alterar layout.

## Riscos
- Nenhum. Injeção via Vite `define` é substituição em compile-time, zero runtime overhead.
- A versão no `package.json` precisa estar em sync com `Cargo.toml` (já é prática do projeto).

**Journal:** ## Key decisions
- **Engine**: Vite `define` instead of a Tauri command — zero runtime overhead, injects `"0.1.5"` at compile time as a global constant.
- **Access pattern**: `APP_VERSION` global constant (not `import.meta.env`) because Vite `define` replaces by name — cleaner in JSX.

## Gotchas
- Tailwind `text-[9px]` is an arbitrary value — works fine with Tailwind v4.
- Vite `define` expects the version to be `JSON.stringify`'d to produce a valid JS string literal in the output.
- The `ml-0.5` (2px) spacing was chosen over `ml-1` (4px) because the middot glyph itself carries visual space — 2px is enough to separate it from "Code" without excess gap.
