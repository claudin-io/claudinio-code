## Context / Problem Statement
A versão atual do Claudinio Code é `0.1.1`. O objetivo é fazer o deploy da tag `v0.1.2`, que dispara o workflow de release via GitHub Actions (`release.yml`), gerando builds para Windows x64, macOS arm64 e Linux x64, e criando um GitHub Release com os binários.

**CONFIRMADO pelo usuário:** Incluir o arquivo `docs/plans/2026-07-11_history-dropdown-click-outside.md` (modificado) no commit de bump.

- **Workflow de release:** `.github/workflows/release.yml` — disparado por `push` de tags `v[0-9]+.[0-9]+.[0-9]+`. Faz build em 3 plataformas, upload de artifacts, e cria GitHub Release.
- **Arquivos a alterar:**

  - `src-tauri/Cargo.toml:2` — `version = "0.1.1"` → `"0.1.2"`
  - `src-tauri/tauri.conf.json` — `"version": "0.1.1"` → `"0.1.2"`
- **Tags remotas existentes:** `v0.1.0`, `v0.1.1` — `v0.1.2` não existe ainda
- **Branch:** `main` (head: `5f5086f`), sincronizada com `origin/main`
- **Working tree:** apenas `docs/plans/2026-07-11_history-dropdown-click-outside.md` modificado

## Solution Design
1. Bump de versão `0.1.1` → `0.1.2` nos 3 arquivos
2. Git add dos 4 arquivos (3 bumps + 1 doc)
3. Commit com mensagem descritiva
4. Criar tag `v0.1.2`
5. Push (commit + tag) para `origin/main`
6. Verificar que o workflow de release foi disparado no GitHub Actions
