# Release v0.1.8

## Context
- Version bumped to 0.1.8 in `package.json` and `Cargo.toml`
- 6 new commits since v0.1.7: auto-update mechanism, auto-updater public key, plus earlier fixes
- Release workflow triggers on `v*` tags
- macOS build failed in earlier runs because:
  1. Secrets were in Environment "RELEASE" but workflow didn't declare `environment: RELEASE` ✅ (fixed in commit `6511268`)
  2. `Resolve Apple Signing Identity` step crashed because `grep "Developer ID Application"` returns exit code 1 with `pipefail`
  3. The imported cert was "Apple Development" not "Developer ID Application" — only good for debug, not distribution
- **Now resolved:** User generated a **Developer ID Application** certificate and updated `APPLE_CERTIFICATE` and `APPLE_CERTIFICATE_PASSWORD` secrets

## Solution Design
1. Push tag `v0.1.8` to origin to trigger the release workflow
2. Monitor the workflow — specifically macOS build
3. Wait for completion and verify release was created on `claudin-io/claudinio-code-releases`

## Risks
- The `Resolve Apple Signing Identity` step still has `set -e pipefail` — if `grep "Developer ID Application"` returns 0 matches, the script crashes before fallback. Current workflow (commit `6511268`) has this issue.
- If needed, fix is `|| true` after the grep — but only if cert lookup still fails

## Non-goals
- Not modifying any other files
- Not touching the source code
- Not making changes to `.env` or configs

## Low-Level Design
- **File to touch:** None (tag only)
- **Action:** `git tag v0.1.8 && git push origin v0.1.8`
- **Monitor:** `gh run list --workflow=release.yml --limit 2`
- **Verification:** Check GitHub Releases page on `claudin-io/claudinio-code-releases` for the v0.1.8 release with macOS `.dmg` and `.app.tar.gz` artifacts
