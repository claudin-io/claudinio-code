**Journal:** ## Key Findings & Decisions

### TypeScript Errors (46 → 0)
- **28 errors** were Vitest globals (`vi`, `afterEach`, `beforeEach`, `setImmediate`) not recognized by TypeScript. Fixed by adding `"types": ["vitest/globals"]` to tsconfig.json and adding missing imports.
- **5 unused declarations** — removed dead code (`accountTier` signal getter) and unused imports.
- **13 property/type mismatches** — Monaco v0.55.1 removed `languages.typescript` namespace, so we removed the deprecated API calls (they were only muting diagnostics — harmless to drop). `DiffViewer.tsx` needed `!` non-null assertions on `containerRef`. `theme.test.ts` type inference was poisoned by `vi.stubGlobal` — fixed by using direct `globalThis` patching.
- `theme.ts` needed `| undefined` on `rootState` variable type.

### Test Coverage
- **8 untested source files** now have test suites. Most complex was `CommitPushModal` (23 tests) due to agent streaming event simulation — solved by capturing the IPC event callback via `vi.hoisted()`.
- `platform.test.ts` already existed (9 tests, 100% coverage) — was just not showing in our initial file listing.
- `FileEditorModal.test.ts` was replaced from `.ts` to `.tsx` to properly render the component instead of only testing pure helper functions — jumped from 13% to 100% line coverage.
- `ipc.ts` coverage went from 81% to 100% by adding tests for 15+ previously-exercised-only-during-runtime functions.
- The remaining branch gaps in `DiffViewer.tsx` and `FileEditorModal.tsx` are V8 coverage dead spots (short-circuit operators, defensive early-returns) — unreachable by design.
