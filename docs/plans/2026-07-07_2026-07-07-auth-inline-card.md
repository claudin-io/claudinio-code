# Auth Inline Card — Replace API Key Error with Authentication Button

## 1. Context / Problem Statement

**CONFIRMED by user:**
- When a user sends a message without authentication (no API key configured), the chat currently shows a raw error text: `"Failed to send: API key not configured. Use set_config first."` as a red chat bubble.
- This UX is unfriendly — it forces the user to discover the config modal, find the login button, sign in, then re-type their message.

**BUG FOUND (2026-07-07, user report):**
- After successful OAuth login, `handleAuthSignIn()` chama `send()`, que por sua vez **adiciona a mensagem original ao array de novo** com `setMessages([...prev, {role:"user", text}])`. Como a mensagem original já está no array desde a primeira tentativa de envio, o resultado é **duplicação da mensagem**.
- Correção: `handleAuthSignIn()` deve chamar `sendMessage()` diretamente (via IPC), sem passar por `send()`.

## 2. Goal (Definition of Done)

Replace the "API key not configured" raw error text with an inline authentication card in the chat stream. After successful login, the card removes itself and the original message is **re-sent WITHOUT duplicating** the message bubble.

Same treatment for `compact_session` auth error.

### Step 6: Implement `handleAuthSignIn()` function — **FIXED for duplication**

**Target:** `src/components/ChatPanel.tsx`, ~line 1033
**Mutation:** 
```typescript
const handleAuthSignIn = async () => {
    setAuthSigningIn(true);
    try {
      await loginWithClaudinio();
      setMessages((prev) => prev.filter((m) => !m.text.startsWith('__auth_card__')));
      const pending = pendingMessage();
      if (pending) {
        setPendingMessage(null);
        // IMPORTANT: do NOT call send() — original message is already in the messages array.
        // Call sendMessage() directly via IPC to avoid duplicating the message bubble.
        setCurrentSteps([]);
        setThinkingStart(0);
        setStatus("thinking");
        scrollToBottom(true);
        try {
          const result = await sendMessage(
            props.workspace,
            pending,
            [],
            handleEvent,
            mode(),
          );
          setActiveSessionId(result.sessionId);
        } catch (e) {
          if (String(e).includes("API key not configured")) {
            setMessages((prev) => [...prev, { role: "user" as const, text: "__auth_card__" }]);
          } else {
            setMessages((prev) => [...prev, { role: "user" as const, text: t("chat.message.failedToSend", String(e)) }]);
          }
          setStatus("idle");
        }
      }
    } catch {
      // Login failed — card stays visible, user can retry
    } finally {
      setAuthSigningIn(false);
    }
  };
```
**Why:** O `send()` original sempre adiciona a mensagem ao array (`setMessages([...prev, {role:"user", text}])`) antes de enviar. Como a mensagem já está no array desde a primeira tentativa, chamar `send()` de novo duplica a bolha. Chamando `sendMessage()` diretamente, só reenviamos ao backend sem tocar no array de mensagens.
**Status:** 🛠️ **Precisa aplicar (Builder mode)**
